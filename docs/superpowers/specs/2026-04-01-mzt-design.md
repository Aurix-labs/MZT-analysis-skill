# MZT (毛泽东思想辩证分析 Skill) 设计文档

## 概述

**项目名称**：MZT (Mao Zedong Thought Dialectical Analysis Skill)

**目标**：创建一个基于毛泽东思想的 AI Agent Skill，提升 Agent 的思考深度和辩证能力，帮助用户精准决策。

**核心理念**：将毛泽东思想中的方法论体系化为可操作的 Agent 能力，通过对话式引导和结构化输出，帮助用户分析问题、制定策略、解决矛盾。

---

## 功能设计

### 双重目标

| 层面 | 描述 | 触发方式 |
|------|------|----------|
| **隐性层** | Agent 思考过程自动融入辩证思维 | `/mzt-on` 开启，`/mzt-off` 关闭 |
| **显性层** | 输出结构化的决策分析报告 | `/mzt` 显式调用 |

### 指令设计

| 指令 | 功能 | 说明 |
|------|------|------|
| `/mzt` | 显式调用分析 | 进入对话式引导，输出结构化报告 |
| `/mzt-on` | 开启辩证思维模式 | Agent 在思考时自动融入辩证方法 |
| `/mzt-off` | 关闭辩证思维模式 | 恢复 Agent 默认思考方式 |

**状态管理**：辩证思维模式状态仅在当前会话有效，会话结束即重置为关闭。

### 对话式引导流程

```
阶段 1: 问题识别
    │
    ▼
阶段 2: 自适应追问（根据问题复杂度决定深度）
    │
    ├── 矛盾层面：核心冲突是什么？
    ├── 客观条件：当前环境、资源、约束？
    ├── 利益相关方：涉及哪些人的利益？
    ├── 时间维度：短期 vs 长期考量？
    └── 风险因素：可能的失败点？
    │
    ▼
阶段 3: 方法论选择（根据问题特征选择合适方法论）
    │
    ▼
阶段 4: 结构化输出
```

### 结构化报告章节

1. **问题定义** — 明确要解决的核心问题
2. **矛盾分析** — 主要矛盾、次要矛盾、矛盾的主要方面
3. **客观条件** — 当前环境和约束条件
4. **解决方案** — 基于分析提出的方案
5. **实践检验** — 如何验证和迭代方案

### 16 个方法论详解

#### 核心哲学层 (4个)

| 序号 | 方法论 | 来源 | 核心要点 | 适用场景 |
|------|--------|------|----------|----------|
| 1 | **矛盾分析法** | 《矛盾论》 | 主要矛盾与次要矛盾、矛盾的主要方面与次要方面、同一性与斗争性 | 复杂问题拆解、优先级判断、冲突分析 |
| 2 | **实事求是法** | 《改造我们的学习》《反对本本主义》 | 从实际出发、反对主观主义、理论联系实际 | 方案评估、现状诊断、问题解决 |
| 3 | **认识循环法** | 《实践论》 | 感性→理性→实践检验；"实践—认识—再实践"循环 | 方案验证、学习改进、迭代优化 |
| 4 | **发展辩证法** | 《星星之火，可以燎原》 | 小与大的转化、弱与强的辩证、长远眼光 | 判断形势、评估潜力、把握趋势 |

#### 分析诊断层 (4个)

| 序号 | 方法论 | 来源 | 核心要点 | 适用场景 |
|------|--------|------|----------|----------|
| 5 | **阶级立场分析法** | 《中国社会各阶级的分析》 | 利益决定立场、"谁是敌人？谁是朋友？"、细分归类 | 利益相关方分析、谈判博弈、市场竞争 |
| 6 | **调查研究法** | 《反对本本主义》 | "没有调查，没有发言权"、调查七法 | 信息收集、现状诊断、决策依据 |
| 7 | **思想纠偏法** | 《关于纠正党内的错误思想》 | 识别错误表现→分析根源→提出纠正方法 | 问题诊断、团队建设、组织管理 |
| 8 | **矛盾分类处理法** | 《关于正确处理人民内部矛盾的问题》 | 敌我矛盾 vs 人民内部矛盾、团结—批评—团结 | 冲突处理、团队管理、问题分类 |

#### 策略执行层 (8个)

| 序号 | 方法论 | 来源 | 核心要点 | 适用场景 |
|------|--------|------|----------|----------|
| 9 | **战略辩证法** | 《论持久战》 | 强弱转化、三阶段论、持久与速决 | 长期规划、竞争策略、资源配置 |
| 10 | **灵活战略法** | 《抗日游击战争的战略问题》 | 保存自己消灭敌人、集中与分散、根据地建设 | 资源有限时的策略选择、灵活应变 |
| 11 | **聚焦突破法** | 《集中优势兵力，各个歼灭敌人》 | 集中资源、重点突破、逐个击破 | 资源分配、优先级执行、项目管理 |

| 13 | **群众路线法** | 《关心群众生活，注意工作方法》 | 从群众中来、到群众中去、解决实际问题 | 需求分析、用户研究、团队管理 |
| 14 | **组织纪律法** | 《反对自由主义》 | 11种自由主义表现、原则性vs人情关系 | 团队协作、组织文化、纪律建设 |
| 15 | **持之以恒法** | 《愚公移山》 | 坚定信念、持续努力、代代相传 | 长期目标、困难任务、信念建设 |
| 16 | **宗旨导向法** | 《为人民服务》 | 根本宗旨、价值导向、意义建构 | 使命愿景、价值观建设、意义感 |

## 方法库文件格式

每个方法论文件遵循以下结构：

```markdown
# [方法论名称]

## 核心原则
- 理论来源（引用毛选原文）
- 核心思想提炼
- 适用场景概述

## 分析框架
- 具体分析步骤
- 关键问题清单
- 分析工具/模板

## 应用示例
- 示例 1：[典型场景]
- 示例 2：[典型场景]

## 详细指导
- 不同场景下的应用技巧
- 常见误区与注意事项
- 与其他方法论的配合使用

## 经典语录
- 毛选中的经典论述摘录
```

**内容详细程度**：每个方法论文件约 1000-1500 字。

## 安装与分发

### 遵循 Agent Skills 规范

本项目遵循 [Agent Skills 规范](https://agentskills.io)，使用 Skills CLI 进行安装分发。

### 安装方式

```bash
# 安装到当前项目的 Claude Code
npx skills add your-github/mzt

# 全局安装到 Claude Code
npx skills add your-github/mzt -g -a claude-code

# 全局安装到 Cursor
npx skills add your-github/mzt -g -a cursor

# 安装到所有支持的 Agent
npx skills add your-github/mzt -g --all
```
