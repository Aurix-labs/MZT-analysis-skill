# 毛泽东思想辩证分析 Skill (MZT)

基于毛泽东思想方法论的 AI Agent 辩证分析能力增强。

## 指令

| 指令 | 功能 |
|------|------|
| `/mzt` | 显式调用分析 — 进入对话式引导，输出结构化报告 |
| `/mzt-on` | 开启辩证思维模式 — Agent 在思考时自动融入辩证方法 |
| `/mzt-off` | 关闭辩证思维模式 — 恢复 Agent 默认思考方式 |

**状态说明**: 辩证思维模式状态仅在当前会话有效，会话结束即重置为关闭。

---

## 核心能力

### 双重目标

1. **隐性层**: 当开启辩证思维模式时，Agent 在思考过程中自动运用辩证方法，提升输出深度
2. **显性层**: 显式调用时，通过对话式引导收集信息，输出结构化的决策分析报告

### 结构化报告章节

当用户使用 `/mzt` 显式调用时，输出包含以下章节：

1. **问题定义** — 明确要解决的核心问题
2. **矛盾分析** — 主要矛盾、次要矛盾、矛盾的主要方面
3. **客观条件** — 当前环境和约束条件
4. **解决方案** — 基于分析提出的方案
5. **实践检验** — 如何验证和迭代方案

## 方法论体系

MZT 包含 16 个方法论，分为三层架构。**必须**根据问题特征选择合适的方法论进行分析。

### 核心哲学层 (Foundation)

世界观与方法论的根基，贯穿所有分析过程。

| 方法论 | 核心要点 | 适用场景 |
|--------|----------|----------|
| [矛盾分析法](methodologies/01-contradiction-analysis.md) | 主要矛盾与次要矛盾、矛盾的主要方面 | 复杂问题拆解、优先级判断 |
| [实事求是法](methodologies/02-seek-truth-from-facts.md) | 从实际出发、反对主观主义 | 现状诊断、方案评估 |
| [认识循环法](methodologies/03-practice-cycle.md) | 感性→理性→实践检验 | 方案验证、迭代优化 |
| [发展辩证法](methodologies/04-development-dialectics.md) | 小与大的转化、长远眼光 | 判断形势、评估潜力 |

### 分析诊断层 (Diagnosis)

用于分析问题、诊断局势、识别关键因素。

| 方法论 | 核心要点 | 适用场景 |
|--------|----------|----------|
| [阶级立场分析法](methodologies/05-class-stance-analysis.md) | 利益决定立场、敌友辨别 | 利益相关方分析、谈判博弈 |
| [调查研究法](methodologies/06-investigation-research.md) | 没有调查没有发言权 | 信息收集、现状诊断 |
| [思想纠偏法](methodologies/07-thought-correction.md) | 识别表现→分析根源→纠正 | 问题诊断、团队建设 |
| [矛盾分类处理法](methodologies/08-contradiction-classification.md) | 敌我矛盾 vs 人民内部矛盾 | 冲突处理、问题分类 |

### 策略执行层 (Execution)

用于制定策略、执行落地、持续优化。

| 方法论 | 核心要点 | 适用场景 |
|--------|----------|----------|
| [战略辩证法](methodologies/09-strategic-dialectics.md) | 强弱转化、持久与速决 | 长期规划、竞争策略 |
| [灵活战略法](methodologies/10-flexible-strategy.md) | 保存自己消灭敌人、灵活应变 | 资源有限时的策略选择 |
| [聚焦突破法](methodologies/11-focus-breakthrough.md) | 集中资源、重点突破 | 资源分配、优先级执行 |
| [统筹兼顾法](methodologies/12-overall-coordination.md) | 十大关系的平衡与协调 | 复杂系统设计、资源平衡 |
| [群众路线法](methodologies/13-mass-line.md) | 从群众中来、到群众中去 | 需求分析、用户研究 |
| [组织纪律法](methodologies/14-organizational-discipline.md) | 原则性vs人情关系 | 团队协作、组织文化 |
| [持之以恒法](methodologies/15-persistence.md) | 坚定信念、持续努力 | 长期目标、困难任务 |
| [宗旨导向法](methodologies/16-purpose-orientation.md) | 根本宗旨、价值导向 | 使命愿景、意义建构 |

## 使用流程

### 显式调用 (`/mzt`)

```
阶段 1: 问题识别
    │
    ▼
阶段 2: 自适应追问（根据问题复杂度决定深度）
    │
    ├── 矛盾层面：核心冲突是什么？
    ├── 客观条件：当前环境、资源、约束？
    ├── 利益相关方：涉及哪些人的利益？
    ├── 时间维度：短期 vs 长期考量？
    └── 风险因素：可能的失败点？
    │
    ▼
阶段 3: 方法论选择（根据问题特征选择合适方法论）
    │
    ▼
阶段 4: 结构化输出（5 个固定章节）
```

### 辩证思维模式 (`/mzt-on`)

当用户开启辩证思维模式后，Agent 在思考时**自动**融入以下原则：

1. **矛盾视角**: 分析问题时，识别主要矛盾和次要矛盾
2. **实践导向**: 方案需要可执行、可验证
3. **立场意识**: 考虑不同利益相关方的立场
4. **发展眼光**: 不仅看当前，还要看长远发展趋势
5. **实事求是**: 基于实际情况分析，避免主观臆断

## 方法论选择指南

根据问题类型选择方法论：

| 问题类型 | 推荐方法论组合 |
|----------|----------------|
| 决策困境 | 矛盾分析法 + 阶级立场分析法 + 聚焦突破法 |
| 战略规划 | 发展辩证法 + 战略辩证法 + 统筹兼顾法 |
| 团队问题 | 思想纠偏法 + 组织纪律法 + 群众路线法 |
| 执行落地 | 聚焦突破法 + 持之以恒法 + 认识循环法 |
| 利益冲突 | 阶级立场分析法 + 矛盾分类处理法 + 群众路线法 |
| 长期目标 | 发展辩证法 + 持之以恒法 + 宗旨导向法 |

---

## 注意事项

1. **方法论是工具，不是教条**: 根据实际问题灵活运用，不可生搬硬套
2. **内容必须来自毛选原文**: 引用经典论述时确保准确性
3. **分析要有深度**: 避免浅尝辄止，要抓住问题本质
4. **输出要有价值**: 提供可执行的建议，而非空泛的理论
