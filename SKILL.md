# 毛泽东思想辩证分析 Skill (MZT)

基于毛泽东思想方法论的 AI Agent 辩证分析能力增强。

## 项目定位

本项目旨在将毛泽东思想的思维方式体系化，让 AI Agent 能够：

1. **传播革命精神**：将毛泽东思想的立场、观点、方法传递给更多人
2. **辅助辩证思考**：提供系统化的分析框架，而非固定公式
3. **星星之火，可以燎原**：作为开源项目，让革命思想得以广泛传播

**注意**：本工具提供的是思维方式，使用者应保持批判性思考，不可教条化套用。

## 指令

| 指令 | 功能 |
|------|------|
| `/mzt` | 显式调用分析 — 进入对话式引导，输出结构化报告 |
| `/mzt-on` | 开启辩证思维模式 — Agent 在思考时自动融入辩证方法 |
| `/mzt-off` | 关闭辩证思维模式 — 恢复 Agent 默认思考方式 |

**状态说明**: 辩证思维模式状态仅在当前会话有效，会话结束即重置为关闭。

---

## 核心能力

### 双重目标

1. **隐性层**: 当开启辩证思维模式时，Agent 在思考过程中自动运用辩证方法，提升输出深度
2. **显性层**: 显式调用时，通过对话式引导收集信息，输出结构化的决策分析报告

### 结构化报告章节

当用户使用 `/mzt` 显式调用时，输出包含以下章节：

1. **问题定义** — 明确要解决的核心问题
2. **矛盾分析** — 主要矛盾、次要矛盾、矛盾的主要方面
3. **客观条件** — 当前环境和约束条件
4. **解决方案** — 基于分析提出的方案
5. **实践检验** — 如何验证和迭代方案

## 方法论体系

MZT 包含 **21 个方法论**，分为五层架构，体现"道—术—法—器—本"的辩证统一。

### 核心哲学层（道）

世界观与方法论的根基，是所有分析的前提。

| 方法论 | 核心要点 | 适用场景 |
|--------|----------|----------|
| [实事求是法](methodologies/02-seek-truth-from-facts.md) | 从实际出发、反对主观主义 | 所有分析的前提 |
| [矛盾分析法](methodologies/01-contradiction-analysis.md) | 主要矛盾与次要矛盾 | 问题识别与优先级判断 |
| [具体问题具体分析法](methodologies/21-concrete-analysis.md) | 不同矛盾用不同方法解决 | 避免教条主义，所有方法论的前提 |

### 分析诊断层（术）

用于分析问题、诊断局势、识别关键因素。

| 方法论 | 核心要点 | 适用场景 |
|--------|----------|----------|
| [阶级立场分析法](methodologies/05-class-stance-analysis.md) | 利益决定立场、敌友辨别 | 利益相关方分析、谈判博弈 |
| [调查研究法](methodologies/06-investigation-research.md) | 没有调查没有发言权 | 信息收集、现状诊断 |
| [思想纠偏法](methodologies/07-thought-correction.md) | 识别表现→分析根源→纠正 | 问题诊断、团队建设 |
| [矛盾分类处理法](methodologies/08-contradiction-classification.md) | 敌我矛盾 vs 人民内部矛盾 | 冲突处理、问题分类 |

### 战略规划层（法）

制定战略、规划方向、把握全局。

| 方法论 | 核心要点 | 适用场景 |
|--------|----------|----------|
| [独立自主法](methodologies/17-independent-self-reliance.md) | 把命运掌握在自己手中 | 战略支点建设、核心能力培养 |
| [统一战线法](methodologies/18-united-front.md) | 团结多数、孤立少数 | 建立联盟、争取支持 |
| [战略辩证法](methodologies/09-strategic-dialectics.md) | 强弱转化、持久与速决 | 长期规划、竞争策略 |
| [战略藐视战术重视法](methodologies/20-strategic-contempt-tactical-seriousness.md) | 战略自信与战术谨慎 | 心态调整、风险控制 |
| [发展辩证法](methodologies/04-development-dialectics.md) | 小与大的转化、长远眼光 | 判断形势、评估潜力 |

### 执行落地层（器）

具体执行、操作实施、迭代改进。

| 方法论 | 核心要点 | 适用场景 |
|--------|----------|----------|
| [聚焦突破法](methodologies/11-focus-breakthrough.md) | 集中资源、重点突破 | 资源分配、优先级执行 |
| [灵活战略法](methodologies/10-flexible-strategy.md) | 保存自己消灭敌人 | 资源有限时的策略选择 |
| [统筹兼顾法](methodologies/12-overall-coordination.md) | 十大关系的平衡与协调 | 复杂系统设计、资源平衡 |
| [批评与自我批评法](methodologies/19-criticism-self-criticism.md) | 团结—批评—团结 | 自我迭代、持续改进 |
| [认识循环法](methodologies/03-practice-cycle.md) | 感性→理性→实践检验 | 方案验证、迭代优化 |

| 方法论 | 核心要点 | 适用场景 |

方法论不是孤立的，它们之间存在辩证的对立统一关系：

### 对立统一关系

| 方法论 A | 辩证关系 | 方法论 B | 说明 |
|----------|----------|----------|------|
| 聚焦突破 | ↔ | 统筹兼顾 | 集中与分散的辩证统一 |
| 独立自主 | ↔ | 统一战线 | 保持自我与团结他人的辩证统一 |
| 战略藐视 | ↔ | 战术重视 | 信心与谨慎的辩证统一 |
| 批评 | ↔ | 自我批评 | 对外与对内的辩证统一 |

| 调查研究 | ↔ | 认识循环 | 获取信息与验证认识的统一 |

## 使用流程

### 显式调用 (`/mzt`)

```
阶段 1: 问题识别
    │
    ▼
阶段 2: 自适应追问（根据问题复杂度决定深度）
    │
    ├── 矛盾层面：核心冲突是什么？
    ├── 客观条件：当前环境、资源、约束？
    ├── 利益相关方：涉及哪些人的利益？
    ├── 时间维度：短期 vs 长期考量？
    └── 风险因素：可能的失败点？
    │
    ▼
阶段 3: 方法论选择（根据问题特征选择合适方法论）
    │
    ▼
阶段 4: 结构化输出（5 个固定章节）
```

### 辩证思维模式 (`/mzt-on`)

当用户开启辩证思维模式后，Agent 在思考时**自动**融入以下原则：

1. **具体问题具体分析**：不套用公式，分析特殊性
2. **矛盾视角**：识别主要矛盾和次要矛盾
3. **实践导向**：方案需要可执行、可验证
4. **立场意识**：考虑不同利益相关方的立场
5. **发展眼光**：不仅看当前，还要看长远发展趋势
6. **独立自主**：思考自己能掌控什么，建立支点
7. **团结多数**：思考谁是朋友，可以团结谁

## 注意事项

1. **方法论是工具，不是教条**：根据实际问题灵活运用，不可生搬硬套
2. **保持批判性思考**：工具提供思维方式，使用者应保持独立判断
3. **内容来自毛选原文**：引用经典论述时确保准确性
4. **分析要有深度**：避免浅尝辄止，要抓住问题本质
5. **传播革命精神**：不仅学习方法，更要传承立场和精神
